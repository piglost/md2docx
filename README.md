# md2docx — Markdown → Word 转换器（法律写作专用）

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)
[![Python 3.10+](https://img.shields.io/badge/Python-3.10%2B-green.svg)](https://www.python.org/)
[![Pandoc 3.0+](https://img.shields.io/badge/Pandoc-3.0%2B-orange.svg)](https://pandoc.org/)

将 Markdown 法学论文/法律文档一键转换为 Word (.docx)，支持《中国法学》《中国社会科学》和课程论文三种格式。

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 📝 **引用→脚注** | `[1]` `[2-5]` `[3,4]` → Word 脚注，正文引用号和脚注编号显示为 `[1]` 方括号样式 |
| 🎨 **精准排版** | 自动检测或手动应用《中国法学》《中国社会科学》或课程论文格式 |
| 🛡️ **内容保真** | 保留加粗、斜体、下划线、链接和列表编号，不改写代码片段 |
| ⬛ **纯黑文字** | 三道防线确保零颜色残留 |
| 📐 **首行缩进** | 正文自动首行缩进 2 字符 |
| 🔤 **格式字体** | 按期刊或课程论文格式分别设置中文字体，英文统一 Times New Roman |

## 📋 格式规范

支持两种期刊格式和一种课程论文格式。期刊格式可自动检测；课程论文格式需手动指定。

### 《中国法学》（`-f faxue`）

| 元素 | 字体 | 字号 | 加粗 | 对齐 |
|------|------|------|------|------|
| 正标题 | 方正小标宋简体 | 22pt | — | 居中 |
| 作者 | 方正楷体简体 | 14pt | — | 居中 |
| 内容提要/关键词标签 | 黑体 | 10.8pt | — | 标签 + 正文同段 |
| 内容提要/关键词正文 | 方正楷体简体 | 10.8pt | 按原文 | 两端对齐、左右缩进 |
| 一级标题 | 黑体 | 14pt | — | 居中 |
| 二级标题 | 黑体 | 10.8pt | — | 左对齐 |
| **正文** | **方正书宋简体** | **10.8pt** | **按原文** | **两端对齐** |
| 脚注 | 方正书宋简体 | 7.8pt | — | 连续编号，段前段后 0 磅 |

### 《中国社会科学》（`-f sheke`）

| 元素 | 字体 | 字号 | 加粗 | 对齐 |
|------|------|------|------|------|
| 正标题 | 方正小标宋简体 | 26pt | — | 居中 |
| 作者 | 方正楷体简体 | 15pt | — | 居中、疏排 |
| 摘要标签 | 方正黑体简体 | 10pt | — | `摘  要` |
| 摘要/关键词内容 | 方正仿宋简体 | 10pt | — | 两端对齐、左右缩进 |
| 一级标题 | 方正小标宋简体 | 15pt | — | 居中 |
| 二、三级标题 | 方正书宋简体 | 11pt | — | 左对齐 |
| **正文** | **方正书宋简体** | **11pt** | **按原文** | **两端对齐** |
| 脚注 | 方正仿宋简体 | 10pt | — | 圆圈数字、每页重编号、段前段后 0 磅 |

### 课程论文（`-f course`）

以《论公共数据开放的数据价值实现原则.docx》提取出的版式为基准。

| 元素 | 字体 | 字号 | 加粗 | 对齐 |
|------|------|------|------|------|
| 正标题 | Heiti SC Light | 18pt | 是 | 居中 |
| 作者/署名 | Songti SC | 12pt | — | 居中 |
| 摘要/关键词标签 | Songti SC | 10.5pt | 是 | 标签 + 正文同段 |
| 摘要/关键词正文 | 仿宋 | 10.5pt | — | 两端对齐、左右缩进 |
| 一级标题 | Heiti SC Medium | 14pt | 是 | 居中 |
| 二级标题 | Heiti SC Medium | 12pt | 是 | 左对齐 |
| **正文** | **Songti SC** | **12pt** | **按原文** | **两端对齐** |
| 脚注 | 宋体 | 9pt | — | 段前段后 0 磅 |

### 段落格式

- 《中国法学》正文：约 541.4 × 754.0 pt 页面，镜像边距，首行缩进约 22.68pt，固定 16.5pt 行距
- 《中国法学》首页不显示页眉；后续奇数页显示文章名，偶数页显示刊名，页脚为外侧页码
- 《中国法学》`--toc` 只列一级标题，并生成实刊样式外框目次
- 《中国社会科学》正文：约 201 × 280 mm 页面、镜像边距、固定 18.6pt 行距
- 《中国社会科学》首页无页眉；后续奇数页显示文章名，偶数页显示刊名，页脚为外侧页码
- 课程论文正文：A4 页面，上下 72pt、左右 90pt，首行缩进 24pt，1.5 倍行距
- 一级标题：居中，段前段后间距
- 如果论文第一部分标题为“引言”或“引  言”，按一级标题格式处理，但保留原文标题，不自动添加“一、”或“二、”编号；没有引言时标题逻辑不变。
- 所有文字颜色：纯黑 `#000000`

## 🚀 快速开始

### 依赖

```bash
brew install pandoc          # macOS
# 或 apt-get install pandoc  # Linux
```

### 安装

```bash
git clone https://github.com/piglost/md2docx.git ~/.claude/tools/md2docx
```

### 作为 Claude Code / Hermes Agent Skill 使用

当前 Claude Code 使用目录式 skill，创建目录链接：

```bash
mkdir -p ~/.claude/skills
ln -s ~/.claude/tools/md2docx ~/.claude/skills/md2docx
```

然后对 AI 说：「把这个 md 转成 word」即可。

### 命令行使用

```bash
python3 md2docx.py 论文.md                    # 生成同名 .docx
python3 md2docx.py 论文.md 输出.docx           # 指定输出文件名
python3 md2docx.py 论文.md --dry-run           # 预览预处理结果
python3 md2docx.py 论文.md --toc               # 插入静态目次（仅中国法学）
python3 md2docx.py 论文.md -f faxue            # 强制《中国法学》格式
python3 md2docx.py 论文.md -f sheke            # 强制《中国社会科学》格式
python3 md2docx.py 论文.md -f course           # 强制课程论文格式
```

### 自动检测逻辑

`--format`（`-f`）参数支持四种值：
- `auto`（默认）：自动检测——①②③ 或 〔1〕 标记 → 中国社会科学；[1] 标记 → 中国法学
- `faxue`：强制《中国法学》实刊格式（方正系列字体、实刊版心、奇偶页眉、外框目次）
- `sheke`：强制《中国社会科学》实刊格式（方正系列字体、镜像版心、奇偶页眉、圆圈脚注）
- `course`：强制课程论文格式（A4、Songti SC 正文、仿宋摘要、无期刊页眉）

## 📂 文件结构要求

```markdown
# 论文标题

**作者：XXX**
**单位：XXX大学法学院**

## 内容提要

内容提要正文段落……

**关键词：** 关键词1；关键词2；关键词3

## 一、一级标题

正文内容，引用使用 [1] 标记。[2][3]

### （一）二级标题

正文内容。更多引用 [4-6]。

## 参考文献

1. 作者A：《文献标题》，出版社2020年版，第10页。
2. 作者B：《文献标题》，载《期刊名》2021年第3期。
```

### 支持的引用格式

| 格式 | 输入 | 输出 |
|------|------|------|
| 单引用 | `[1]` | 脚注 `[1]` |
| 连续多引用 | `[1][2][3]` | 脚注 `[1]`、`[2]`、`[3]` |
| 范围引用 | `[1-5]` | 脚注 `[1]`-`[5]` 展开 |
| 逗号引用 | `[1,3,5]` | 脚注 `[1]`、`[3]`、`[5]` |

### 参考文献条目格式

支持四种格式：
- `[1] 作者：《标题》，出版社，年份。`（方括号）
- `1. 作者：《标题》，出版社，年份。`（数字句号）
- `〔1〕作者：《标题》，出版社，年份。`（全角方括号）
- `① 作者：《标题》，出版社，年份。`（圆圈数字，支持 ①-⑳）

参考文献条目支持缩进续行。正文中的行内代码、围栏代码、数字链接和图片链接不会被误识别为引用。引用编号缺失、重复定义、逆序范围或超过 100 项的范围会中止转换并报告具体错误。

## 🔧 转换流水线

```
Markdown (.md)
    │
    ▼
[1] 预处理：引用标记 → Pandoc 脚注语法
    │
    ▼
[2] Pandoc 转换：md → docx + strip-color.lua 去色
    │
    ▼
[3] Heading 修复：移除 Pandoc 蓝绿标题残留
    │
    ▼
[4] 格式后处理：按 faxue/sheke/course 设置字体、字号、对齐、缩进和脚注
    │
    ▼
Word (.docx) ✅
```

所有阶段先写入临时文件。Pandoc 或格式化失败时命令返回非零状态，不会用半成品覆盖已有输出文件。
